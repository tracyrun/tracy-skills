# 可行性研究报告编写

## 核心流程

收到用户请求后，按以下流程执行：

### 第一步：收集项目基本信息

从用户输入中提取，缺失项主动询问：

| 必填项    | 说明               |
| ------ | ---------------- |
| 项目名称   | 正式全称             |
| 项目地址   | 省/市/县，越具体越好      |
| 建设单位   | 全称               |
| 初步建设内容 | 对建设内容的初步描述       |
| 资金来源   | 财政资金还是自筹，有无贷款或债券 |

### 第二步：按推荐顺序逐章编写

可研报告的编写遵循**先论证定位，再技术定案，后算账定钱，最后汇总**的逻辑。推荐顺序：

| 阶段        | 顺序 | 章节                   | 回答的核心问题                       |
| --------- | -- | -------------------- | ----------------------------- |
| **基础论证**  | ①  | **第2章** 项目建设背景和必要性   | 为什么要建？                        |
| <br />    | ②  | **第3章** 项目需求分析与产出方案  | 建多大？产出什么？                     |
| <br />    | ③  | **第4章** 项目选址与要素保障    | 建在哪里？要素够不够？                   |
| **技术方案**  | ④  | **第5章** 项目建设方案       | **具体怎么建？**（技术/设备/工程/数字化/管理方案） |
| **运营与财务** | ⑤  | **第6章** 项目运营方案       | 怎么运营？                         |
| <br />    | ⑥  | **第7章** 项目投融资与财务方案 ⭐ | **投多少钱？钱从哪来？赚不赚钱？**           |
| **影响与风险** | ⑦  | **第8章** 项目影响效果分析     | 对经济/社会/环境有什么影响？               |
| <br />    | ⑧  | **第9章** 项目风险管控方案     | 有什么风险？怎么防？                    |
| **汇总收尾**  | ⑨  | **第1章** 概述           | 汇总前面所有章节的核心结论                 |
| <br />    | ⑩  | **第10章** 研究结论及建议     | 给出最终结论和建议                     |
| <br />    | ⑪  | **第11章** 附表、附图和附件    | 随各章编写同步整理                     |

按此顺序向用户展示，从第2章开始编写。用户也可指定从任意章节开始。

### ⚠️ 关键依赖关系（必须遵守）

| 前序章节    | 后续章节    | 依赖关系            | 违规风险                     |
| ------- | ------- | --------------- | ------------------------ |
| **第5章** | **第7章** | 建设方案是投资估算的技术依据  | ❌ 先估后建 → 投资额缺乏技术支撑，逻辑不自洽 |
| **第3章** | **第5章** | 建设规模决定技术方案工程量   | 工程量与需求脱节                 |
| **第6章** | **第7章** | 运营方案决定运营成本和收益估算 | 财务评价缺乏运营依据               |

**规则**：

- 如用户要求跳过依赖章节先写后续章节，必须明确提示风险
- 如第7章在第5章之前编写，则第7章所有估算标注为 **"初步匡算（待第5章确定后复核修正）"**，待第5章完成后回核修正

### 第三步：逐章编写循环

对每一章执行以下子流程：

1. **依赖检查**（每章开始前执行）：
   - 检查本章的依赖章节是否已完成（见上方依赖关系表）
   - 如依赖章节未完成且用户要求跳过：**主动提示风险**，说明可能产生的逻辑问题
   - 如用户仍要求跳过：在生成的章节开头标注 ⚠️ 风险提示，说明所缺失的前序章节
2. **联网研究**（每章首次进入时执行）：
   - 搜索相关政策法规（标注发布机构、文号、生效日期）
   - 搜索同类项目案例（建设规模、投资额、工期）
   - 搜索行业标准和规范（标准编号、名称）
   - 搜索当地市场数据（造价指标、材料价格等）
   - 要求：至少 3 个独立来源交叉验证
3. **第7章专属：融资方案前置确认**（仅第7章执行）：
   - 在开始第7章编写前，先向用户确认融资结构关键要素：
     - 资金来源及金额（政府拨款/中央补助/专项债/银行贷款/自筹）
     - 债务期限和利率
     - 是否涉及专项债作资本金
   - 确认后再开始估算，避免写完返工
   - 如用户回答"不确定"，先用常见方案编写，标注为 **"初步方案（待确认）"**
4. **生成初稿**：
   - 先查阅 `references/detailed-outline-template.md` 中该章的逐节写作提示，按模板结构展开
   - 严格按照 `references/chapter-writing-guide.md` 中该章的深度和字数要求编写
   - 优先采用段落化叙述（自然段），避免过度使用列表列点
   - 所有数据标注来源
   - 不确定的地方用 `【待确认：...】` 标注
   - 估算类数据说明估算依据和方法
5. **用户审核**：
   - 展示初稿，询问：确认通过 / 需要修改 / 跳过此章
   - 修改时：接受具体修改意见，调整后再次确认
6. **反向复核**（章审通过后执行）：
   - 检查是否有之前已完成但依赖于本章的章节（如第5章完成后，检查第7章是否已写）
   - 如有，提示用户："第X章依赖本章数据，需要回核修正"
   - 执行回核修正，并记录修正内容
7. **继续或结束**：
   - 用户选择下一章，或结束本次编写

### 第四步：整合输出

所有章节完成后：

1. **最终融资方案确认**：与用户做最后一次融资方案确认（是否还有调整），如有调整，执行上方"融资方案变更传播机制"
2. **重新排序**：将章节按大纲标准顺序（第1章→第2章→……→第11章）排列
3. **反向复核检查**：检查第5章和第7章的数据一致性（见 `references/writing-standards.md` 中的"第5章-第7章依赖校验"表），如第7章先于第5章编写，必须执行回核修正
4. **生成 Excel 文件**：从第7章分项估算表生成 `[项目名称]-投资估算表.xlsx`
5. **合并报告**：使用 Bash 脚本合并所有章节子文件为单一完整报告 `[项目名称]-可研报告-完整版.md`，合并时注意：
   - 添加封面
   - 子文件按章节内部逻辑顺序拼接
6. **一致性检查**：执行 `references/writing-standards.md` 中的质量检查清单，重点核对：
   - 第1章汇总数据与各对应章节是否一致
   - 投资估算 Excel 与第7章文本是否一致
   - 各章项目名称、建设规模等数据是否统一
   - 第5章技术方案的投资与第7章估算是否匹配
   - 用 Grep 搜索已删除的敏感词（如银行融资项目删除政府资金、中央预算投资、专项债券，资金申报项目删除贷款、替换景观、美化、楼堂馆所等负面清单描述），确保零残留
7. **输出**：Markdown 完整报告 + Excel 投资估算表文件

## 核心原则

1. **内容专业准确**：数据必有来源，估算必说明方法，不确定必标注
2. **格式规范工整**：严格按发改委2023大纲结构输出
3. **逐章可控**：每章都需用户确认后才进入下一章
4. **联网研究贯穿始终**：每章都需要最新数据支撑
5. **自然段叙述优先**：正文内容优先采用段落化叙述，避免过度使用列表列点。以下场景必须使用自然段：
   - 第1章 主要结论（三自然段，硬性要求）
   - 第2章 规划政策符合性（每个规划/政策对应一个自然段）
   - 第2章 项目建设必要性（每条必要性一个自然段）
   - 各章的分析论证内容
6. **经营性/非经营性分支**：编写各章时必须根据项目性质选择对应路径（详见 `references/detailed-outline-template.md` 附录"项目类型分支速查"），关键分支点：
   - 第1章：经营性项目需写法人和组建方案，非经营性无需
   - 第3章：经营性项目需做竞品分析和SWOT，非经营性分析服务对象
   - 第5章：按项目类型（建筑类/道路类/其他）选择工程方案子项
   - 第7章：经营性项目算FIRR/FNPV/Pt，非经营性做全生命周期资金平衡

## ⚠️ 第7章特别说明：建议预留独立会话

### 为什么第7章需要特殊对待

第7章（项目投融资与财务方案）是本报告**最复杂、上下文消耗最大**的章节，原因：

1. **数据密集**：投资估算表含数十项分项工程，每项需标注单位数量、单价指标、合价、备注，表格本身即占据大量上下文
2. **计算链条长**：投资估算→融资结构→成本收益→盈利能力→偿债能力→财务可持续性，六段连环，每段都依赖前段数据
3. **跨章依赖多**：第7章同时依赖第5章（技术方案决定工程量）、第6章（运营方案决定运营成本）、第3章（建设规模决定体量），任何一章变更都会传导至第7章
4. **融资变更频繁**：用户常在编写过程中调整融资方案（取消专项债、调整银行贷款、取消中央补助等），每次变更需同步更新第1/7/9/10章及投资估算表，涉及6\~8个文件

**实测经验**：本项目编写中，第7章及其后续融资方案调整合计消耗了约60%的会话上下文，远超其他章节。

### 建议做法

| 做法               | 说明                                                                         |
| ---------------- | -------------------------------------------------------------------------- |
| **独立会话编写第7章**    | 在前期章节（第2\~6章）完成后，开新会话专写第7章，确保上下文窗口充足                                       |
| **先确认融资方案再动笔**   | 在第7章开始前，先与用户确认：有无专项债？有无中央补助？银行贷款额度？避免写完后大幅返工                               |
| **融资变更走传播表**     | 如编写过程中融资方案变更，严格按照下方"融资方案变更传播机制"执行，逐文件更新，防止遗漏                               |
| **后续可建专用 Skill** | 如项目类型固定（如物流基地/市政道路/标准厂房），可单独建立 `feasibility-ch7-finance` Skill，封装融资模板和计算逻辑 |

> 💡 **提示**：当用户说"继续写第七章"时，如果当前会话已消耗大量上下文（已完成4\~6章），主动建议："第7章计算量大、文件多，建议开新会话专门编写，确保计算准确。是否继续？"

## 融资方案变更传播机制

当用户要求修改融资方案（如取消专项债、调整银行贷款、取消中央补助、变更资本金比例等），**必须按以下传播表逐文件更新**，不得遗漏：

### 变更传播矩阵

| 变更类型        |    第1章    |     第7章完整版    | 第7章子文件 |    第9章    |   第10章   | 投资估算表.md |  合并报告  | 其他可能影响    |
| ----------- | :-------: | :-----------: | :----: | :-------: | :------: | :------: | :----: | --------- |
| 取消/调整专项债    |   ✅ 资金表   | ✅ 7.3/7.4/7.5 | ✅ 同完整版 | ✅ R12/R13 |  ✅ 10.2  |  ✅ 建设期利息 | ✅ 重新生成 | 第2/3章政策引用 |
| 调整银行贷款额度/利率 |   ✅ 资金表   |   ✅ 7.3/7.4   | ✅ 同完整版 | ✅ R12/R13 |     —    |  ✅ 建设期利息 | ✅ 重新生成 | —         |
| 取消/调整中央补助   |   ✅ 资金表   |     ✅ 7.3     | ✅ 同完整版 | ✅ R14（删除） |  ✅ 10.2  |     —    | ✅ 重新生成 | 第3章SWOT   |
| 变更资本金比例     |   ✅ 资金表   |    ✅ 7.3.1    | ✅ 同完整版 |     —     | ✅ 10.1.5 |     —    | ✅ 重新生成 | —         |
| 变更建设单位名称    | ✅ 1.1/1.2 |    ✅ 7.3.2    | ✅ 同完整版 |     —     |     —    |     —    | ✅ 重新生成 | 第2/6章     |

### 变更执行流程

1. **确认变更范围**：根据上表确定受影响的所有文件
2. **先改源文件**：按第1章→第7章（完整版+子文件）→第9章→第10章→投资估算表→其他章节的顺序修改
3. **重新计算财务指标**：偿债备付率(DSCR)、利息备付率(ICR)、加权融资成本、年还本付息额、建设期利息
4. **更新风险登记表**：如R12/R13/R14涉及融资，同步更新第9章风险识别/评估/管控三张表，并调整风险总数
5. **全文搜索验证**：用 Grep 搜索旧融资关键词（如"专项债""中央预算内"），确保零残留
6. **重新生成合并报告**：所有源文件修改完毕后，重新运行合并脚本

## 第7章编写要点：项目投融资与财务方案

第7章涵盖投资估算和财务分析两大块，必须做到**分项详细估算**。

### 投资估算粒度要求

每项工程/费用必须拆解到：

- **单位数量**：工程量（㎡、m³、台、套等）
- **单位指标**：综合单价（元/㎡、元/m³、元/台等）
- **合价**：单位数量 × 单位指标

### Excel 输出要求

完成分项估算后，用 Python 生成 `.xlsx` 文件：

- 第1个Sheet：**投资估算表** — 上表结构 + 表头加粗 + 合计行底纹
- 第2个Sheet：**编制说明** — 记录估算依据、参考来源、取费标准
- 第3个Sheet（如有）：**财务评价表** — 现金流量表、利润表
- 文件名：`[项目名称]-投资估算表.xlsx`
- 金额列数字格式：`#,##0.00` 万元

## 估算能力

基于初步建设内容，Skill 应主动进行合理估算：

- **投资估算**：分项详细估算（见上），参考同类项目单位造价指标 + 当地市场价格
- **建设工期**：参考同类项目 + 行业标准工期定额，在第5章建设管理方案中确定
- **用地规模**：参考相关用地标准 + 建设规模反推，在第4章中确定
- **劳动定员**：参考行业标准 + 项目规模，在第6章运营组织方案中确定

所有估算必须说明依据、方法和参考来源。
